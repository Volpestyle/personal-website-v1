NPR Audio Engineering Discrepancy Logger

MEAN Stack (JavaScript)

I spent the summer of 2019 in Washington D.C., interning for National Public Radio. The Audio Engineering Discrepancy Logger, or AED Logger for short, was my project for the entirety of the summer. It was a way for NPR employees to manage a log of technical difficulties encountered during audio production. Before my time at NPR, the system in place to log incidents was running on an outdated tech-stack and wasn't as elegant as many of the users would have liked. It was my job to redesign this system with up-to date technology and to contain more features.

Some of the main features included:

  • Automatic account creation for NPR employees.
  • Searching for logs that fit various criteria.
  • Log creation.
  • Personal subscriptions that send email or text notifications to a specific user when a log that fits certain critiera is submitted.
  • Global subscription that many users can subscribe to at once
  • Automatic I.T. ticket creation.
  • User roles (Administrative abilites, ability to create global subscription, etc.)

The full documentation of my implementation that I shared with my team can be found in pdf form inside this github repo.

Parse Tree Calculator

Vanilla JavaScript

This is a side-project I did not only to improve my general JavaScript skills but to also specifically learn more about data structures, classes, and event-handlers in JS. It was created completely from scratch in HTML, SASS, and Vanilla JavaScript. This calculator works by tokenizing the given expression, creating a tree structure accordingly, and evaluating the tree with a recursive method. This way, the expression can be evaluated in the correct order of operations.

Roomba Mars Rover

Embedded Systems | C

(The above image is not mine and belongs to this Youtuber user)

This was my final project for CPR E 288: Intro to Embedded Systems. This was a group project that I completed with my team members Daksh Goel, Jordan Fox, and Steven Beard.

Together, we formed our initial project idea and proposed it to our Professor and TA's for approval. We decided on a project that would enable our Roomba to travel across a field of obstacles to a position specified by our TA. As it did this, it would determine the position of each obstacle and display it on a primitive GUI. We would then use this GUI to travel across the field without actually having to see our robot.

The goal of this project was to merge each functionality gained from our labs into one final product. For example, in two separate labs, we learned how to use the Roomba's ultrasonic sensor to determine the distance of objects and how to use pulse width modulation to move the servo motor attached to the sensor. While creating our final project, we used these two functionalities to make our sensor scan 180 degrees to the front, detecting the obstacles and returning their positions back to us.
